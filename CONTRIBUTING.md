public-safe documentation, installation, extension, and source-catalog work.
Reallexi LLC maintainers decide what is accepted, revised, deferred, declined,
or moved to a separately controlled profile.

## Contribution License

By submitting a contribution, you confirm that you have the right to submit it
and license it under Apache-2.0. Reallexi LLC may use, modify, distribute,
commercialize, or decline the contribution under the project governance rules.

Do not submit confidential material, credentials, customer data, private model
weights, downloaded datasets, or third-party work you cannot license.

## Good Public Contributions

* clearer installation and troubleshooting documentation;
* public policy, support, security, and accessibility improvements;
* training-source metadata and license corrections;
* plugin or theme manifest proposals that use documented public categories;
* issue templates, examples, and launcher validation improvements;
* reproducible reports that contain synthetic data only.

Changes to application implementation, internal operations, access enforcement,
or private integrations should be proposed as feature requests. Do not attach
private source or reverse-engineered implementation details.

## Pull Request Steps

1. Search existing issues and pull requests.
2. Keep one contribution focused on one public concern.
3. Use synthetic examples and public links only.
4. Run `npm run validate`.
5. Run `npm run setup` when installer behavior changes.
6. Explain the user impact and any license or data-provenance considerations.

The public repository is generated from a controlled source of truth. When a
public contribution is accepted, maintainers carry it into that source and
regenerate the public package so later releases do not overwrite it.
