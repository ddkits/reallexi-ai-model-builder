# Security Policy

Do not report suspected vulnerabilities in public issues.

## Reporting

Email security reports to info@reallexi.io. If GitHub private vulnerability
reporting is enabled for this repository, you may use that instead.

Include:

* affected version or commit;
* reproduction steps;
* impact;
* logs or screenshots if safe to share;
* whether any secret, token, model weight, dataset, or private data is involved.

## Supported Versions

The latest public community release receives best-effort security review.
Private or commercial deployments require a separate Reallexi LLC support
agreement.

## Handling

Reallexi LLC will triage reports, decide whether the issue affects the community
or private profile, and publish fixes or guidance when appropriate.
