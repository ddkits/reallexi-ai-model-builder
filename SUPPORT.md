# Support

Community support is best effort.

Use GitHub issues for:

* reproducible bugs;
* documentation gaps;
* community feature requests;
* setup and validation problems.

Do not use public issues for:

* vulnerabilities;
* secrets or credentials;
* private datasets;
* customer-specific deployment details;
* paid support requests.

Commercial support, managed operations, private features, and enterprise
integrations require a separate Reallexi LLC agreement.
